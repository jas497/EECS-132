Programming Project 1 Grading Rubric

For the grading, the getter/setter methods are all methods that only set a value or return a value. These are the methods that start with get or set, but also include deposit, transfer of the Cash class.

1. Testing: 20 points

PointsMetric
20 A description of how to thoroughly test all the methods of the assignment. Completed tests for each method written in the student's submission. Each test gives the true output of the student's submission. A note on any test that gives an incorrect result.
18 A description of how to thoroughly test all the methods of the assignment and completed tests for all the non-getter/setter methods written in the student's submission, all the constructors are tested, and at least three quarters of the getter/setter methods are tested. Each test gives the true output of the student's submission. At most two cases where the testing report fails to make note of a case where the test demonstrates an error in the student's code.
15 A description on how to thoroughly at least half of the 36 getter/setter methods and at least seven of the 13 non-getter/setter methods. All tests accurately represent the student's code.
12 Includes reasonable tests of at least 10 methods including at least five of the 13 non-getter/setter methods. The results of tests accurately represent the student's submission.
10 Includes a reasonable test of all the methods one class. The results of tests accurately represent the student's submission.
0 Does not include a reasonable test for all methods of any one of the classes OR up to two tests do not accurately represent how the student's code works.
-10 Includes output for methods that were not in the student's submission OR gives at least three results misrepresenting how the student's code works.

2. Readability: 20 points

PointsMetric
20 Correct comments at top of class file, above each method and and above each field. Each comment at most one or two sentences and is a good description. Class has fields first, then constructors, then the remaining methods. All code follows the proper indentation and naming style. (See page 689 of the recommended textbook for a style suggestion.) There is a blank line between each method.
18 Mostly indented well (up to two places not indented correctly) AND most names correctly follow the naming convention (up to two poorly named variables or methods) AND coments on most methods, constructors, fields, and the top of the file (up to four missing comments). Each comment is a good description.
15 More than two places of poor indenting OR more than two method or variable names that do not follow the Java convention OR more than four missing comments OR comments are longer than one or two sentences OR comments are direct copies of the homework description.
12 Comments missing from at least half of the places that require them OR all of the if statements are poorly indented OR at least half of the methods are poorly indented
5 Missing comments in the code OR poor indentation throughout the code
0 No comments in the code AND poor indentation in the code.

3. Correctness: 60 points

3.a. Fields: 10 points

PointsMetric
10All fields provided. All fields private. All fields are given explicit initial values either at the declaration or in the constructor. No extraneous fields. No static fields or methods.
9Most of the fields are done correctly. Less than half of the fields have problems and the is only one type of problem with the fields: some not provided, some not private, some not given explicit initial values, some are static OR some extraneous fields provided.
7Most of the fields are done correctly. Less than half of the fields have problems and there are up to three different types of problems with the fields: some not provided, some not private, some not given explicit initial values, some are static or some extraneous fields provided.
5At least one class has at least half of the necessary fields provided, and at least one field is private, non-static, and has an explicit initial value.
0No class has at least half of the necessary fields provided OR every field has at least two of the following problems: is static, not private, and does not set an initial value.

3.b. Getter and setter methods: 20 points

PointsMetric
20 All public getter/setter methods provided and correct (public, getter correctly gets the value, setter correctly sets the field, no extra processing done), no extra public getter or setter methods not listed in the assignment. (Extra private methods are ok.)
18 All getter/setter methods correct, but extra public getter or setter methods provided that are not listed in the assignment OR missing the getter-setter methods (if the methods are required in the assignment) for up to 3 fields.
15 Missing the getter/setter methods for more than 3 fields, but all provided getter/setter methods are correct OR all getter setter methods provided but up to 6 cases where the setter method fails to set the field or the getter method fails to return the proper value or the method is not public or the method does extra processing.
10 At least 3 cases where the provided getter/setter methods correctly return the proper value and set the proper field.
5 Provides at least one correct getter/setter pair for some field.
0 No correct getter or setter methods provided.

3.c. Constructors: 10 points

PointsMetric
10 All constructors correct.
9 The constructors fail to set the value for one field OR the constructors work correctly but have extra unnecessary operations or duplicate computation.
7 At least one class has a correct constructor.
5 All constructors fail to properly set field values OR the methods are not really constructors because they are incorrectly given return values.
0 No constructors provided.

3.d. Non-getter/setter Methods: 20 points

PointsMetric
20 All methods are correct.
18 All methods that require conditional statements have well-formed statements for the problem. There could possibly be minor errors in the mathematical processing. All the methods that do not require conditional statements are correct or have only minor errors.
15 At least six of the methods that require conditional statements have well-formed statements for the problem with possibly minor errors in the mathematical processing.
10 At least three of the methods that require conditional statements have well-formed statements for the problem, possibly with minor errors.
5 At least one method has an appropriatly formed conditional statement for the problem with possible minor errors OR all the methods that do not require conditional statements have only minor errors.
0 No method has an appropriate conditional statement for the problem AND at least one of the methods that does not require a conditional statement has major errors.
