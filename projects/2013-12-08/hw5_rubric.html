Programming Project 5 Grading Rubric

1. JUnit Testing: 20 points

PointsMetric
20 Correct JUnit tests demonstrate that the code (including the linked list) correctly works ignoring random numbers or user/file inputs. Tests of conditionals cover the different execution paths. Tests of loops correctly test 0, 1, many and first, middle, last. Report clearly describes the kind of tests needed, and matches JUnit tests written. Descriptions of required tests for incomplete methods and descriptions of code errors are included.
18 Similar to 20 except that a few obvious cases were not tested.
15 Correct JUnit tests for all methods, possibly missing just a few obvious cases, but no testing report provided OR a good testing report provided as well as mostly complete tests for at least three classes including the EmployeeDatabase class.
12 At least once class has all necessary JUnit tests completed.
10 At least a few methods that have conditional statements or loops have correct JUnit tests written.
0 No reasonable JUnit testing done.

2. Readability: 20 points

Basic program structure

PointsMetric
10 Correct comments at top of class, above each method, above each field, above each loop, and next to or above each variable. (Comments are -not- required next to any index variables used in the loops.) The comments are at most one to three sentences and are a good description. The comments above each loop indicate how the loop works and any prerequisites for the loop. (The loop subgoal/invariant would be nice but is -not- required.) All code follows the proper indentation and naming style. An empty line is placed between the end of one method and the comment of the next.
9 Mostly indented well (up to two places not indented correctly) AND most names correctly follow the naming convention (up to two poorly named variables or methods) AND at most one missing comment on a loop or other complicated code. Each comment is a good description.
8 Good indentation for at least three of the methods AND at least half of the methods and loops have comments that are at most 1-3 sentences and not direct copies of the homework description.
6 At least one method is well commented or there is a comment above every method and the class AND at least one method is nicely indented.
3 At least one method is well commented OR at least one method is nicely indented.
0 No methods have either good comments or good indentation.

JavaDoc Comments

PointsMetric
10All comments above the class, fields, and methods follow the JavaDoc comment style. Correct tags are added for the author, all method parameters and return types.
9All comments above the class, fields, and methods follow the JavaDoc comment style, and most of the comments contain and have correct all required tags. (At most 5 missing or misspelled tags.)
7Most comments above the class, fields, and methods follow the JavaDoc comment style, but a significant number of places are missing tags or the tags are misspelled so the comments are not formatted properly on the webpage.
5Most comments in JavaDoc style of /** but missing most tags OR has most tags but the comments do not start with /** and so do not appear on the webpage.
0No JavaDoc comments provided.

3. Correctness: 60 points

PointsMetric

18There is an error in ONE of the position in the heirarchy OR the use of the enum OR the Comparable interface OR the equals method OR the other methods and constructor. There are no additional methods provided.
16All three elements: name, health status, and relations have correct getter/setter methods and constructor initialization. At least ONE of the enum type, the Comparable interface, and the equals method are implemented correctly. There are no additional methods provided.
14All three elements: name, health status, and relations have correct getter/setter methods and constructor initialization. There are no additional methods provided.
10All three elements: name, health status, and relations have correct getter/setter methods and constructor initialization, but additional methods are added to the Entity class beyond the ones indicated.
0If the class is implemented, there are at most two elements with correct getter/setter methods and constructor initialization.

PointsMetric

18The same as a 20 answer but there are only minor errors.

PointsMetric

18The class uses the Network iterator correctly, the Entity Comparable interface is used correctly to order the entities when printing, the main methods is mostly correct, there are no major errors but either the main method throws an exception OR the enum type is not used correctly OR the class is not in the proper position in the hierarchy OR some of the methods were not completed.
